# Process

## Current process

1. Research scene families
2. Generate or select plate candidates
3. Choose a daily winner or approved branch
4. Apply native-artifact mapping
5. Mount into live scaffold
6. Review in:
   - live
   - clickable
   - solo
   - debug
7. Seed with recent Obsidian signals
8. Add ambience only after mapping and readability work
9. Publish daily edition

## Native-artifact mapping rule

Masks must follow actual visible artifacts in the plate.
Good targets:
- specimen bodies
- label slips
- notes
- instruments
- cards
- object clusters
- candles
- diagrams
- titles
- icon clusters

Bad targets:
- empty space
- arbitrary quadrants
- generic compositional zones
- fake dashboard modules

## Recovery lessons

If the page starts feeling wrong:
- check whether the scene is backdrop instead of interface
- check whether hotspots are arbitrary overlay zones
- check whether chrome is overpowering the plate
- fix manifestation after geometry if the architecture is right but the look is still wrong

## Current review contract

Step 1: `?debug=masks`
- geometry truth check only

Step 2: `?qa=clickable`
- clean clickability check

Step 3: `?qa=solo`
- isolate one region at a time

Step 4: `?review=current-live`
- live behavior check
